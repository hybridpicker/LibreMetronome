Developer Guidelines

CSS Naming Conventions

LibreMetronome follows a modified BEM (Block, Element, Modifier) naming convention:

  • Block: Standalone component (e.g., metronome, beat, controls)
  • Element: Parts of a block (e.g., metronome__indicator, controls__button)
  • Modifier: Variations of blocks or elements (e.g., beat--accent, button--primary)

Component Organization

Keep CSS organized by following these principles:

  • Use component-specific CSS files
  • Import global variables from a central location
  • Avoid !important tags
  • Use CSS variables for themes and variants
  • Keep specificity low by avoiding deep nesting

Accessibility Guidelines

Ensure all components meet accessibility standards:

  • Maintain color contrast ratios (at least 4.5:1 for normal text)
  • Provide focus states for interactive elements
  • Include appropriate ARIA attributes
  • Ensure keyboard navigability
  • Test with screen readers
